================================================================
ALUR PEMAKAIAN
================================================================

A) PIHAK KAMPUS — Mendaftarkan ijazah
   Tab "Kampus — Daftarkan":
   1. Pilih file PDF ijazah (polos, tanpa cap).
      - Belum punya? Klik "Buat PDF Contoh".
   2. Isi: Kode Unik (mis. UGM-010203-HUDZAIFAH), Nama, NIM.
   3. Klik "Daftarkan & Amankan".

   Sistem otomatis:
   - Memberi CAP visual berisi kode unik pada PDF.
   - Menyisipkan METADATA + TANDA TANGAN DIGITAL (HMAC-SHA256).
   - MENGUNCI PDF dengan password = kode unik (AES-256 native PDF).
   - Mencatat SHA-256 file ke BLOCKCHAIN (inti C++).

   Output: file  <nama>_SECURED.pdf
   -> Berikan file ini ke mahasiswa. File HANYA bisa dibuka
      dengan kode unik (diminta saat dibuka di Adobe/Chrome/dll).


B) PUBLIK / MAHASISWA — Memverifikasi
   Tab "Verifikasi":
   1. Pilih file *_SECURED.pdf.
   2. Masukkan Kode Unik.
   3. Klik "Verifikasi Sekarang".

   Hasil:
   - HIJAU  "IJAZAH ASLI"  -> hash cocok + kode benar (Double-Lock lolos),
            tampil Nama, NIM, waktu daftar.
   - MERAH  "TIDAK VALID"  -> file diubah / kode salah / tak terdaftar.


================================================================
KENAPA AMAN (Double-Lock)
================================================================
  Lock 1 (Integritas): SHA-256 file. Ubah 1 byte -> verifikasi GAGAL.
  Lock 2 (Kepemilikan): kode unik dienkripsi AES-256 di blockchain.
                        Plus: PDF terkunci, tak bisa dibuka tanpa kode.
  Immutability: blockchain linked-hash. Ubah block lama -> chain rusak.

  Cek integritas chain kapan saja: tombol "Cek Integritas Blockchain".
